1. **Переменные нужны для безопасности и переиспользования**. Склейка строк опасна из-за SQL-инъекций (точнее GraphQL-инъекций) и неудобна - нельзя переиспользовать запрос с разными данными.

2. **Стандартный JSON-запрос содержит**:
   - `query` или `mutation` - строка с запросом
   - `variables` - объект с переменными (опционально)
   - `operationName` - имя операции (если несколько операций в запросе)

3. **Один URL** потому что GraphQL - это единая точка входа (endpoint). Вся логика маршрутизации происходит внутри запроса, а не на уровне URL.

4. **Поле `errors`** означает, что в процессе выполнения запроса произошли ошибки. Да, оно может прийти вместе с `data` - частичный успех (например, одно поле запроса упало, другое выполнилось).

5. **Клиент должен** проверять наличие поля `errors` в теле ответа, даже если статус 200. HTTP 200 значит только "сервер ответил", но не "запрос выполнен успешно".

6. **Плюсы типизированных клиентов**:
   - Автодополнение кода
   - Валидация запросов на этапе компиляции
   - Кэширование результатов
   - Автоматическое обновление UI при изменении данных

## 1. Разница между Forward Proxy и Reverse Proxy

**Forward Proxy (обычный прокси):**
- Работает **на стороне клиента**
- Клиенты знают о существовании прокси
- Скрывает клиента от сервера
- Используется для обхода блокировок, кеширования, анонимности
- Пример: клиент в корпоративной сети выходит в интернет через прокси

**Reverse Proxy (обратный прокси):**
- Работает **на стороне сервера**
- Клиенты не знают о существовании прокси
- Скрывает серверы от клиента
- Используется для балансировки нагрузки, обеспечения безопасности, SSL termination
- Пример: Nginx перед микросервисами (как в вашем задании)

## 2. Что такое Upstream в терминологии Nginx?

**Upstream** — это группа бэкенд-серверов, на которые Nginx перенаправляет запросы. Определяет пул серверов, между которыми можно распределять нагрузку.

```nginx
upstream shipments_service {
    server host.docker.internal:8270;
    server host.docker.internal:8271; 
}
```

## 3. Зачем скрывать реальные IP-адреса сервисов от клиента?

1. **Безопасность:** Злоумышленник не может напрямую атаковать сервисы, только Gateway
2. **Гибкость инфраструктуры:** Можно менять IP-адреса, добавлять/удалять сервисы без изменения клиентского кода
3. **Единая точка входа:** Клиенту нужно знать только один адрес
4. **Балансировка нагрузки:** Gateway сам распределяет запросы между экземплярами

## 4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway?

1. **Аутентификация и авторизация:** Проверка JWT-токенов, API-ключей до передачи запроса к сервисам
2. **Rate limiting (ограничение скорости запросов):** Защита от DDoS-атак и чрезмерной нагрузки
3. **Кеширование ответов:** Уменьшение нагрузки на бэкенд
4. **Логирование и мониторинг:** Сбор метрик со всех запросов в одном месте
5. **Трансформация протоколов:** Например, преобразование REST в gRPC
6. **SSL termination:** Расшифровка HTTPS-трафика

## 5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

**Последствия падения Gateway:**
- Все сервисы становятся недоступными для клиентов
- Single Point of Failure (единая точка отказа)

**Защита:**
1. **Репликация Gateway:** Запуск нескольких экземпляров за балансировщиком
2. **DNS-балансировка:** Несколько IP-адресов для одного домена
3. **Health checks:** Автоматическое обнаружение и переключение на рабочий экземпляр
4. **Использование orchestration tools:** Kubernetes автоматически перезапускает упавшие контейнеры
5. **Резервирование:** Active-Passive или Active-Active конфигурации
